# Engineering Checklists

소프트웨어 개발 과정에서 반복적으로 확인해야 하는 항목을 정리한 실무용
체크리스트 모음입니다. 각 문서는 그대로 복사해 사용하거나 프로젝트 상황에
맞게 필요한 항목만 선택할 수 있도록 작성합니다.

## 체크리스트

- [Pull Request 검토](docs/pull-request-review.md): 변경 범위, 정확성, 테스트,
  보안과 운영 영향을 병합 전에 확인합니다.

## 사용 방법

1. 작업 성격에 맞는 체크리스트를 엽니다.
2. 해당하지 않는 항목은 제외하고 프로젝트별 항목을 추가합니다.
3. 확인되지 않은 위험은 Pull Request에 근거와 후속 조치를 기록합니다.

## 작성 원칙

- 각 항목은 확인 여부를 명확하게 판단할 수 있도록 작성합니다.
- 형식 준수보다 결함과 운영 위험을 조기에 발견하는 것을 우선합니다.
- 기술 용어는 원문을 유지하고 필요한 경우 한국어 설명을 함께 표기합니다.
- 특정 언어나 프레임워크에 종속된 내용은 별도 문서로 분리합니다.
