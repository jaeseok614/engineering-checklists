# Release 준비 및 배포 체크리스트

서비스 변경을 운영 환경에 배포하기 전후로 검증해야 할 항목을 정리합니다.
변경 위험도에 따라 필요한 항목을 선택하고, 담당자와 판단 근거를 Release 기록에
남깁니다.

## Release 범위

- [ ] 배포 대상 커밋, 태그와 artifact(배포 산출물)가 일치한다.
- [ ] 이번 Release에 포함된 기능, 수정 사항과 제외 사항이 명확하다.
- [ ] 사용자와 연동 시스템에 영향을 주는 변경을 식별했다.
- [ ] 배포 담당자, 승인자와 장애 대응 담당자가 정해져 있다.
- [ ] 고객 안내나 점검 공지가 필요한지 확인했다.

## 코드와 자동 검증

- [ ] 기본 브랜치의 필수 Continuous Integration(CI) 검사가 통과했다.
- [ ] 단위, 통합 및 핵심 사용자 흐름 테스트를 실행했다.
- [ ] 실패하거나 건너뛴 테스트의 이유와 위험을 검토했다.
- [ ] 의존성 취약점과 라이선스 검사 결과를 확인했다.
- [ ] 빌드가 깨끗한 환경에서 재현 가능하다.
- [ ] 배포 artifact의 checksum 또는 식별자를 기록했다.

## 설정과 비밀값

- [ ] 환경별 설정값의 추가·변경·삭제 내역을 확인했다.
- [ ] 비밀값은 Secret Manager 등 승인된 경로로 전달한다.
- [ ] 새 환경변수에 기본값, 유효성 검사와 문서가 있다.
- [ ] 기능 플래그(feature flag)의 초기 상태와 담당자가 정해져 있다.
- [ ] 로그와 오류 메시지에 민감한 값이 출력되지 않는다.

## 데이터와 호환성

- [ ] 데이터베이스 migration(스키마 변경)의 실행 순서가 명확하다.
- [ ] 구버전과 신버전 애플리케이션이 함께 실행되는 동안 호환된다.
- [ ] 대용량 migration의 잠금, 실행 시간과 저장 공간 영향을 검토했다.
- [ ] 변경 전 백업 또는 복구 지점을 확인했다.
- [ ] 데이터 변환을 재실행해도 중복이나 손상이 발생하지 않는다.
- [ ] 되돌릴 수 없는 데이터 변경에는 별도 승인과 복구 계획이 있다.

## 배포 계획

- [ ] 배포 순서와 예상 소요 시간이 문서화되었다.
- [ ] Rolling, Canary 또는 Blue-Green 등 배포 방식을 선택했다.
- [ ] 트래픽이 적은 시간대가 필요한지 확인했다.
- [ ] 외부 API, 메시지 큐와 배치 작업의 배포 순서를 확인했다.
- [ ] 배포 명령은 대상 환경과 버전을 명시적으로 확인한다.
- [ ] 중단 시 다시 시작할 checkpoint(재개 지점)가 있다.

## 관찰과 검증

- [ ] 배포 전 정상 상태의 주요 지표를 기록했다.
- [ ] 오류율, 응답시간, 처리량과 자원 사용량을 관찰할 수 있다.
- [ ] 핵심 사용자 흐름에 대한 smoke test(기본 동작 검사)가 준비되었다.
- [ ] 경고 임계값과 알림 수신자가 올바르게 설정되었다.
- [ ] 로그, 메트릭과 분산 추적에서 Release 버전을 식별할 수 있다.
- [ ] 기술 지표뿐 아니라 주요 비즈니스 지표의 이상 여부도 확인한다.

## 되돌리기 기준

- [ ] rollback(되돌리기)을 시작할 오류율과 지연시간 기준이 정해져 있다.
- [ ] 이전 정상 artifact와 설정을 즉시 사용할 수 있다.
- [ ] 애플리케이션과 데이터베이스를 되돌리는 순서가 명확하다.
- [ ] 기능 플래그로 위험 기능만 비활성화할 수 있는지 확인했다.
- [ ] 되돌리기 명령과 권한을 배포 전에 검증했다.
- [ ] rollback이 불가능한 경우 트래픽 차단 또는 읽기 전용 전환안이 있다.

## 배포 후 확인

- [ ] 배포된 버전과 커밋이 계획한 값과 일치한다.
- [ ] smoke test와 주요 모니터링 지표가 정상이다.
- [ ] 오류 로그와 고객 문의에 새로운 이상 징후가 없다.
- [ ] 임시 기능 플래그와 우회 설정의 후속 작업을 등록했다.
- [ ] Release 결과, 실제 소요 시간과 발생한 문제를 기록했다.
- [ ] 장애나 예상 밖 상황에서 얻은 개선점을 체크리스트에 반영한다.

## Go/No-Go 기록

배포 직전에는 아래 정보를 남겨 판단 책임과 근거를 명확히 합니다.

| 항목 | 기록 내용 |
| --- | --- |
| Release 버전 | 태그, 커밋 또는 artifact 식별자 |
| 배포 환경 | 대상 서비스와 환경 |
| 담당자 | 배포, 승인 및 장애 대응 담당자 |
| 확인된 위험 | 남아 있는 위험과 완화 조치 |
| 관찰 지표 | 정상 여부를 판단할 대시보드와 임계값 |
| rollback 기준 | 되돌리기를 시작할 구체적인 조건 |
| 최종 판단 | Go, 조건부 Go 또는 No-Go와 근거 |
